\section{Introduzione} \label{intro}

\subsection{Obiettivo del documento}
L’obiettivo del presente \glossario{documento} è descrivere in modo rigoroso le funzionalità e l'utilizzo del prodotto \textit{DIPReader}.\\
L’analisi guiderà l'utente attraverso il contesto applicativo e le procedure operative per una gestione corretta del sistema.

\subsection{Panoramica del prodotto}
\textit{DIPReader} è un'applicazione progettata per l'accesso e la visualizzazione di documenti informatici all'interno di un Dissemination Information Package (\glossario{DIP}). Un DIP è un archivio compresso, distribuito da un sistema di \glossario{conservazione} centralizzato, che contiene documenti tecnici con valore legale e strutturati secondo specifiche normative. Poiché la struttura interna di questi pacchetti è complessa e difficilmente consultabile tramite strumenti generici, \textit{DIPReader} fornisce una soluzione dedicata per esplorare, ricercare e consultare tali archivi in modo efficiente.

\subsection{Utenti di destinazione}
Il prodotto costituisce una soluzione essenziale per gli utenti operanti nei settori giudiziario e tecnico. Tra i principali destinatari figurano l'Agenzia delle Entrate, la Guardia di Finanza e i magistrati, i quali necessitano di consultare i complessi documenti digitali normati e presenti nei pacchetti DIP.

\section{Requisiti di Sistema} \label{requisiti}

\subsection{Requisiti minimi di sistema}
Per garantire un corretto funzionamento, il dispositivo ospite deve soddisfare i seguenti requisiti minimi:
\begin{itemize}
    \item \textbf{RAM}: Almeno 4 GB.
    \item \textbf{Archiviazione}: Almeno 1 GB di spazio libero su disco.
    \item \textbf{Processore}: Quad-core 64-bit a 2.0 Ghz o superiore (architettura x64 o ARM64).
\end{itemize}

\subsection{Sistemi Operativi supportati}
Il sistema garantisce l'esecuzione sui seguenti sistemi operativi (o equivalenti/superiori):
\begin{itemize}
    \item Windows 10.
    \item MacOS 11.
    \item Linux Ubuntu 20.04.
\end{itemize}

\subsection{Modalità di esecuzione}
Il software opera in totale modalità \textit{portable}. Questo significa che l'applicazione non richiede alcuna installazione sul disco del dispositivo né privilegi di amministratore nel sistema ospite per poter funzionare, favorendo flessibilità e immediatezza di utilizzo.

\section{Panoramica delle Funzionalità} \label{panoramica}
Le principali funzionalità offerte da \textit{DIPReader} per l'interazione con i pacchetti DIP includono:
\begin{itemize}
    \item \textbf{Navigazione e Visualizzazione:} Esplorazione gerarchica (Classe documentale, Processo, Documento) all'interno del DIP e visualizzazione in anteprima dei formati di documenti supportati.
    \item \textbf{Ricerca Avanzata e Semantica:} Ricerca all'interno dei metadati tramite filtri specifici avanzati e ricerca semantica dei documenti.
    \item \textbf{Verifica Integrità:} Controllo dello stato di verifica (Integro/Non Integro/Non Verificato) dei file contenuti nel DIP e generazione dei relativi report.
    \item \textbf{Estrazione ed Esportazione:} Possibilità di scaricare singolarmente o in blocco i documenti sul computer locale, stampare i file tramite le periferiche di sistema o esportare i report di verifica in PDF.
    \end{itemize}

Si verrà reindirizzati a questa sezione nel momento in cui si seleziona un nodo dell'albero di navigazione (sezione \ref{esplorazione ad albero}) oppure se si seleziona un risultato della ricerca (sezione \ref{ricerca}). \subsection{Esplorazione ad albero} \label{esplorazione ad albero}
In qualsiasi momento è possibile vedere a sinistra una sezione dedicata alla vista ad albero del file DIP. Ogni riga rappresenta un raggruppamento oppure un file. Se una riga presenta una tabulazione, allora appartiene alla prima riga sopra che non presenta l'indentazione. La gerarchia dei raggruppamenti è la seguente:
\begin{itemize}
    \item Il nodo radice DIP contiene le classi documentali
    \item Le classi documenti contengono i processi associati (pallino rosso)
    \item I processi, che sono numerati in modo ordinato, contengono i file principali (pallino verde)
    \item I file principali contengono gli allegati (pallino azzurro) 
\end{itemize}
È possibile navigare questa sezione tramite il TAB. Se si desidera è possibile chiudere questa sezione tramite l'apposita linguetta "Albero" adiacente alla sezione.

\subsection{Stampa e Salvataggio Documentale} \label {esport}

L'utente può stampare o estrarre in locale i documenti conservati nel sistema. Il software supporta la stampa o il salvataggio sia del singolo file principale, che di interi pacchetti documentali.

\subsubsection{Procedura di Esportazione}

L'azione di stampa è analoga a quella d'esportazione, dunque è mostrata solo l'esportazione. L'esportazione è disponibile esclusivamente per i nodi di tipo \textit{Documento Padre}. Per avviare il processo, seguire i passaggi indicati:

\begin{enumerate}
    \item \textbf{Selezione del Documento}: Identificare nell'albero di navigazione il documento desiderato. Una volta selezionato, l'area centrale mostrerà i metadati e le azioni disponibili;
    \item \textbf{Attivazione Comando}: Cliccare sul pulsante \textbf{Scarica documento} (contrassegnato dal colore verde) situato nella barra superiore delle azioni (Figura \ref{fig:barra_azioni});
\end{enumerate}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{img/barra_azioni.png}
    \caption{Barra delle azioni disponibili per i documenti padre.}
    \label{fig:barra_azioni}
\end{figure}

\begin{enumerate}[resume]
    \item \textbf{Configurazione dello scaricamento}: Il sistema aprirà una finestra di dialogo modale (\textit{Seleziona i documenti da scaricare}) che elenca il file principale e tutti gli eventuali allegati correlati (Figura \ref{fig:modale}).
\end{enumerate}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.7\textwidth]{img/modale.png}
    \caption{Interfaccia di selezione granulare dei file per l'esportazione.}
    \label{fig:modale}
\end{figure}
\subsubsection{Selezione Multipla e Allegati}

All'interno della finestra di selezione, l'utente ha il pieno controllo sugli elementi da esportare:
\begin{itemize}
    \item \textbf{Selezione Granulare}: È possibile spuntare o deselezionare singolarmente ogni file. Per impostazione predefinita, il sistema seleziona l'intero pacchetto (file principale + allegati);
    \item \textbf{Etichette di Tipologia}: Accanto a ogni nome file è presente un'etichetta identificativa (\textit{Principale} o \textit{Allegato}) per facilitare il riconoscimento dei contenuti;
    \item \textbf{Conferma}: Il pulsante di conferma indica tra parentesi il numero totale di file pronti per lo scaricamento.
\end{itemize}

\subsubsection{Monitoraggio e Conclusione}

Una volta avviata l'operazione, il software gestisce il trasferimento dei dati tramite il layer di infrastruttura IPC, fornendo feedback in tempo reale:

\begin{itemize}
    \item \textbf{Avanzamento}: In alto a destra comparirà una barra di stato che mostra l'elenco dei file in fase di scrittura e la conferma individuale per ogni elemento completato;
    \item \textbf{Notifica Finale}: Al termine dell'intera procedura, un messaggio di conferma indicherà il numero totale di documenti salvati con successo sul \glossario{file system} locale (Figura \ref{fig:notifica_export});
\end{itemize}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.6\textwidth]{img/notifica.png}
    \caption{Messaggi di feedback per la conferma del salvataggio.}
    \label{fig:notifica_export}
\end{figure}
\begin{itemize}
    \item \textbf{Destinazione}: I file vengono salvati nella cartella di destinazione predefinita del sistema o in quella selezionata tramite la finestra di dialogo ministeriale, mantenendo i nomi originali dei file presenti nel DIP.
\end{itemize}
