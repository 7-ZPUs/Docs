\section{Backend – Export e Stampa}

\subsection{Panoramica}

Il modulo di export e stampa gestisce due flussi distinti ma correlati: l'esportazione di file dal pacchetto DiP verso il filesystem locale dell'utente, e la stampa di uno o 
più file tramite il sistema di stampa nativo del sistema operativo.

L'orchestrazione è gestita da due Use Case dedicati: \textit{ExportFileUC} e \textit{PrintFileUC}. Entrambi ricevono un identificativo numerico del file da elaborare, 
recuperano la relativa entità tramite \textit{IFileRepository}, risolvono il percorso assoluto all'interno del pacchetto DiP e delegano l'operazione alle rispettive port 
outbound (\textit{IExportPort} e \textit{IPrintPort}).

Il punto di ingresso lato Electron è \textit{FileViewerIpcAdapter}, che registra i canali IPC su \textit{ipcMain} e funge da ponte tra il processo renderer e i Use Case 
del backend. Per le operazioni che richiedono interazione con l'utente (scelta del percorso di destinazione, conferma della stampa multipla) ci si affida a un'ulteriore port
 outbound, \textit{IDialogPort}, che astrae le finestre di dialogo native del sistema operativo.

\subsubsection{FileViewerIpcAdapter}

\begin{figure}[H]
    \centering

    \caption{Diagramma delle classi – FileViewerIpcAdapter, Use Case e ExportResult}
    \label{fig:export_ipc_part1}
\end{figure}

\textit{FileViewerIpcAdapter} è l'adapter IPC che registra i canali di comunicazione tra il processo renderer e il processo main di Electron, esponendo le funzionalità di 
export e stampa al frontend. Risolve le dipendenze \textit{IExportFileUC}, \textit{IPrintFileUC} e \textit{IDialogPort} tramite il container DI \textit{tsyringe}.

\paragraph{Metodo principale}
\textit{+ register(ipcMain: IpcMain): void} – metodo statico che registra tutti i canali IPC del modulo.

\paragraph{Canali registrati}

\begin{itemize}
    \item \textit{FILE\_PRINT} – avvia la stampa di un file dato il suo \textit{fileId}, delegando a \textit{PrintFileUC.execute()}.
    \item \textit{FILE\_DOWNLOAD} – avvia l'esportazione di un file dato \textit{fileId} e \textit{destPath}, delegando a \textit{ExportFileUC.execute()}.
    \item \textit{FILE\_SAVE\_DIALOG} – apre la finestra di salvataggio nativa tramite \textit{IDialogPort.showSaveDialog}, restituendo il percorso scelto dall'utente o il flag \textit{canceled}.
    \item \textit{FILE\_FOLDER\_DIALOG} – apre la finestra di selezione directory tramite \\\textit{IDialogPort.showFolderDialog}, restituendo il percorso della cartella selezionata o 
    il flag \textit{canceled}.
\end{itemize}

\subsubsection{Use Case}

\paragraph{ExportFileUC}

Use Case che orchestra il processo di esportazione di un file dal pacchetto DiP verso il filesystem locale. Recupera l'entità \textit{File} tramite \textit{IFileRepository}, ne 
risolve il percorso assoluto e delega la scrittura su disco a \textit{IExportPort}.

\textit{+ execute(fileId: number, targetPath: string): Promise<ExportResult>} : avvia l'esportazione del file identificato da \textit{fileId} verso \textit{targetPath}. Se il
 file non viene trovato nel repository, restituisce un \textit{ExportResult} di fallimento con codice \textit{NOT\_FOUND}.

\paragraph{PrintFileUC}

Use Case che orchestra la stampa di un singolo file. Recupera l'entità \textit{File} tramite \textit{IFileRepository}, ne risolve il percorso assoluto all'interno del pacchetto DiP 
e delega la stampa a \textit{IPrintPort}.

\textit{+ execute(fileId: number): Promise<ExportResult>} : avvia la stampa del file identificato da \textit{fileId}. Se il file non viene trovato nel repository, restituisce un 
\textit{ExportResult} di fallimento con codice \textit{NOT\_FOUND}.


\subsubsection{ExportResult}

Value object che rappresenta l'esito di un'operazione di export o stampa, contenendo le informazioni necessarie a diagnosticare eventuali errori.

\begin{itemize}
    \item \textit{+ success: boolean} : indica se l'operazione è stata completata con successo.
    \item \textit{+ errorCode?: string} : codice identificativo dell'errore, presente solo in caso di fallimento.
    \item \textit{+ errorMessage?: string} : messaggio descrittivo dell'errore, presente solo in caso di fallimento.
    \item \textit{+ static ok(): ExportResult} : factory method che restituisce un'istanza di successo.
    \item \textit{+ static fail(errorCode: string, errorMessage: string): ExportResult} : factory method che restituisce un'istanza di fallimento con il codice e il messaggio forniti.
\end{itemize}

\subsubsection{LocalExportPort e PrintPort}

\paragraph{LocalExportPort}

\begin{figure}[H]
    \centering
    \includegraphics[width=1\textwidth]{../../../Docs/out/3_PB/DocumentiEsterni/DiagrammiSpecificaTecnica/C4/C4_backend_Export/Export_IPC_Part3.png}
    \caption{Diagramma delle classi – Infrastructure Layer: adapter e port}
    \label{fig:export_ipc_part3}
\end{figure}

Adapter outbound che implementa \textit{IExportPort}, responsabile della scrittura del contenuto di un file su disco tramite le API di streaming di Node.js.

\textit{+ exportFile(stream: NodeJS.ReadableStream, destPath: string): Promise<ExportResult>} : riceve uno stream leggibile e lo scrive nel percorso di destinazione tramite 
\textit{createWriteStream}. In caso di errore, restituisce un \textit{ExportResult} di fallimento con codice \textit{WRITE\_ERROR}.

\paragraph{PrintPort}

Adapter outbound che implementa \textit{IPrintPort}, responsabile della stampa di un singolo file tramite le API native di Electron. Crea una \textit{BrowserWindow} nascosta, carica il 
file tramite il protocollo \textit{file://} e invoca la stampa al termine del rendering.

\textit{+ printSingle(absolutePath: string, opts: WebContentsPrintOptions): Promise<ExportResult>} : carica il file nel percorso indicato all'interno di una finestra Electron nascosta e
 avvia la stampa con le opzioni fornite. Terminata l'operazione, restituisce un \textit{ExportResult} che ne descrive l'esito.

\paragraph{IDialogPort}

Interfaccia outbound che astrae le interazioni con l'utente che richiedono finestre di dialogo native del sistema operativo. Separa la logica di business dell'adapter IPC 
dall'implementazione concreta basata sulle API di Electron.

\begin{itemize}
    \item \textit{+ showSaveDialog(opts: SaveDialogOptions): Promise<SaveDialogResult>} : mostra la finestra di salvataggio nativa con le opzioni fornite (es.\ filtri estensione,
     percorso predefinito). Restituisce il percorso scelto dall'utente oppure un flag \textit{canceled} se l'operazione viene annullata.
    \item \textit{+ showFolderDialog(): Promise<FolderDialogResult>} : mostra la finestra di selezione di una directory. Restituisce il percorso della cartella selezionata oppure
     un flag \textit{canceled} se l'operazione viene annullata.
    \item \textit{+ showConfirmPrint(fileCount: number): Promise<ConfirmPrintResult>} : mostra una finestra di conferma prima di avviare una stampa multipla, informando l'utente
     del numero di file coinvolti. Restituisce l'esito della scelta dell'utente (confermato o annullato).
\end{itemize}

\paragraph{ElectronDialogPort}

Adapter outbound che implementa \textit{IDialogPort} utilizzando le API native di Electron. Fa da ponte tra la logica applicativa e il modulo \textit{dialog} di Electron,
delegando a \textit{dialog.showSaveDialog}, \textit{dialog.showOpenDialog} e \textit{dialog.showMessageBox} rispettivamente per le tre operazioni esposte dall'interfaccia.
La finestra di riferimento per i dialog è ricavata tramite \textit{BrowserWindow.getFocusedWindow()} al momento dell'invocazione.
