\section{Backend - Browsing}

\subsection{Panoramica}
\label{sec:browsing}

\noindent
Il modulo Browsing implementa la navigazione gerarchica dei dati indicizzati in SQLite: 
DIP\ped{G} $\to$ Classi documentali $\to$ Processi $\to$ Documenti $\to$ File. 
Supporta inoltre getter alternativi per ID e stato di integrità.


\noindent
Le caratteristiche backend trasversali al container Main (stack tecnologico, layering, sequenza runtime generale, Dependency Injection, bootstrap e sicurezza preload) sono descritte in \nameref{sec:container:main}.


\subsubsection{Flusso end-to-end}
\label{sec:browsing:flusso}

\paragraph{Flusso gerarchico di navigazione.}
\begin{center}
  \codeword{BROWSE\_GET\_DIP\_BY\_ID} \\
  $\downarrow$ \\
  \codeword{BROWSE\_GET\_DOCUMENT\_CLASS\_BY\_DIP\_ID} \\
  $\downarrow$ \\
  \codeword{BROWSE\_GET\_PROCESS\_BY\_DOCUMENT\_CLASS} \\
  $\downarrow$ \\
  \codeword{BROWSE\_GET\_DOCUMENTS\_BY\_PROCESS} \\
  $\downarrow$ \\
  \codeword{BROWSE\_GET\_FILE\_BY\_DOCUMENT}
\end{center}

Contratto IPC definito in \codeword{ipc-channels.ts} e registrato in \codeword{BrowsingIpcAdapter.ts}:

\subsubsection{Comportamenti runtime}
\label{sec:browsing:runtime}

\noindent
\textbf{By-ID:} restituisce \codeword{null} se entity non trovata\\
\textbf{List:} restituisce array, anche vuoto\\
\textbf{DTO:} conversione applicata nel boundary IPC prima della restituzione

\paragraph{Lettura file buffer (\codeword{BROWSE\_GET\_FILE\_BUFFER\_BY\_ID}).}
\begin{enumerate}[labelwidth=0.8cm,leftmargin=1cm]
  \item Recupera file entity tramite \codeword{IGetFileByIdUC}
  \item Risolve percorso assoluto: \codeword{path.resolve(dipPath, file.getPath())}
  \item Legge contenuto con \codeword{fs.readFileSync}
  \item In caso di errore I/O: log e ritorno \codeword{null}
\end{enumerate}
